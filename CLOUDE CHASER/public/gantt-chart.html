Note:  Each iteration follows six subphases — Planning, Analysis, Design, Development, Testing, Evaluation.
Methodology: The study employed an iterative Agile-inspired methodology consisting of three iterations spanning six months. Each iteration followed the same six subphases — Planning, Analysis, Design, Development, Testing, and Evaluation — allowing the developers to incrementally deliver functionality, gather feedback from stakeholders (Records Office, Deans, VPAA, VPREI, and the University President), and refine the system before proceeding to the next iteration. Iteration 1 established the core foundation (authentication, user management, basic travel requests). Iteration 2 implemented the workflow layer (invitations, endorsement letters, multi-level approvals, Travel Orders). Iteration 3 introduced real-world features (digital signatures with QR verification, expense reconciliation, budget code validation, and liability waivers).
